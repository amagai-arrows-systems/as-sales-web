# Git操作に関するAI活用ガイドライン

1. 目的

本ガイドラインは、Git操作の効率化と品質向上、およびセキュリティリスクの回避を目的としたAI（Gemini）活用のための最低限のルールを定義する。

2. セキュリティ・機密情報の保護

* 機密情報の除外: APIキー、パスワード、接続文字列、個人情報を含むコードは絶対に送信しない。
* コードの抽象化: 秘匿性の高いビジネスロジックを相談する場合、変数名や関数名を汎用的な名称に置き換えた「疑似コード」を使用する。

3. コミットメッセージの作成規約
コミットメッセージの生成を依頼する際は、以下のルールを遵守させる。

* 形式: Conventional Commitsに準拠する。
* 形式例: <type>(<scope>): <subject>
* 構成:
1. 1行目：要約（50文字以内）
   2. 2行目：空行
   3. 3行目以降：変更理由や内容の詳細（箇条書き）
* インプット: git diff --cached の出力結果を渡し、差分に基づいた正確な記述を求める。

4. Gitコマンド提案の制限

* 破壊的操作の警告: 以下のコマンドを含む提案を行う際は、必ず実行前のリスクと代替案を併記させる。
* git push --force
   * git reset --hard
   * git rebase (共有ブランチに対して)
* 安全確認: 可能な限り --dry-run オプションを付けた確認手順を提示させる。

5. プルリクエスト（PR）の作成・レビュー

* 構成案の作成: 変更内容を要約し、「背景」「変更内容」「影響範囲」「テスト内容」をMarkdown形式で出力させる。
* セルフレビュー: PR作成前に、コードの可読性、バグの可能性、命名規則の違反がないかAIにチェックを依頼する。

6. 透明性の確保

* AI生成の明示: AIによって生成された複雑なスクリプトや、AIが代筆したPR説明文には、末尾に (Generated by Gemini) と記載する。
